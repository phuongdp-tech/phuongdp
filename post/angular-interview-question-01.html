1. So sánh TypeScript và JavaScript?

# JavaScript TypeScript
Definition JavaScript là một programming language phổ biến cho cả front-end và back-end

Font-end: trên các browser environment, javascript giúp tạo ra những website có tính tương tác cao.

Back-end: trên node.js environment, javascript có thể làm những việc tương tự như một server-side programing language(Java, Python, PHP...)

 TypeScript là một programing language được xây dựng dựa trên JavaScript. Có thể hiểu rằng:

TypeScript = JavaScript + More features
Typing Loosely typed: Chỉ support dynamic type
Data type của biến linh động phụ thuộc vào data biến đó lưu trữ 		Strongly typed: Support cả dynamic type và static type
Error Detection Errors chỉ xác định được tại thời điểm run time, vì nó là một ngôn ngữ interpreted language. Errors có thể xác định sớm tại thời điểm compile.
Execution JavaScript có thể execute trực tiếp trên môi trường browser và node.js TypeScript cần phải compile thành JavaScript sau đó mới thực thi được trên môi trường browser và node.js

2. Angular Module là gì

Angular application là một ứng dụng được thiết kế dạng module hóa. Mỗi module là một container chứa các source code xử lý logic của những chức năng có cùng business hoặc cùng tầng bảo mật.

Mỗi Angular application có một root module (AppModule) và một hoặc nhiều feature modules.

Mỗi Angular module sẽ được định nghĩa với một @NgModule decorator với 5 thông tin quan trọng như sau:

Properties Detail
declarations Sử dụng để khai báo các components, directives, pipes thuộc về module này.
exports Là tập hợp con của declarations, sử dụng để khai báo các components, directives, pipes có thể được sử dụng trong component templates của các module khác.
imports Sử dụng để import các dependent modules khác.
Ví dụ: BrowserModule là module required cho bất kỳ web based angular application nào.
providers Sử dụng để configure tập hợp các đối tượng có thể inject trong module này (Có thể là service, interceptor, global config...)
bootstrap Config component nào sẽ khởi động Angular application. Chỉ root module config bootstrap property

3. Angular Component là gì?

Trong Angular, mỗi component sẽ control một phần UI hiển thị trên màn hình (a patch of screen). Nó bao gồm 3 phần cơ bản:

  • TypeScript class chứa các xử lý business logic cho component.
  • HTML template chứa thông tin view cho component.
  • CSS style sheet định nghĩa style look and feel cho component.

Mỗi component sẽ được định nghĩa với một @Component decorator:

Properties Detail Example
selector

Định nghĩa tag name đại diện cho component khi muốn insert component này vào template.

 
@Component({
  selector:    'app-my-component',
  templateUrl: './my-component.component.html'
})

Insert component này vào template HTML sử dụng: <app-my-component></app-my-component>

File template HTML của component này: my-component.component.html
templateUrl Chỉ định file template cho component, ngoài ra có thể dùng template property để cung cấp trực tiếp HTML inline.
